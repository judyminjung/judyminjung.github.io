Sal Eats

DATE:

Spring 2020

DESCRIPTION:

This is a course project from University of Southern California Viterbi School of Engineering CSCI 201 course (Spring 2020).

This is a web application that allows users to sign-up and login or authenticate through Google Sign-in. The program allows users to search for restaurants by passing in restaurant name, sort option (best match, rating, review count, distance), and coordinates to the Yelp API, add them to their list of favorites, and schedule reservations for these restaurants on their personal Google calendars.

I used MySQL database to store the data of the users and their favorited restaurants. The design was a simple three table design: UserRegistry, RestaurantRegistry, and Favorites table. Each registry table held the appropriate information regarding the specified element such as login information for user and restaurant address and contact for the restaurant. The favorites table held two foreign keys, one specifying the user and the other their favorited restaurant.

Note worthy: A previous version of this web application read in data from a hard-coded text file and used the Yelp API to retrieve restaurant data. This version simulated a multithreaded program running on localhost through a server and a client. It was a networked delivery system where clients (which represent drivers) receive orders from the server (which represents the headquarters), and the drivers will deliver the orders to the appropriate restaurants.

Technologies:

HTML/CSS, JavaScript, Java, JSP, MySQL, Eclipse, Tomcat

APIs:

Google Sign-in, Google Calendar, Google Maps, Yelp Business Search
